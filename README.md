# System-Design Study Project

A self-contained **system-design study project** wired to the
[Multi-knowledgeGraph](https://github.com/) engine. You drop in lecture slides
and a video; Claude becomes your **陪讀教練 (study coach)** — it reads the PDF,
watches the video via Gemini, discusses the material with you, and writes what
you learn into a persistent knowledge graph (`system-design.db`) that you can
later merge into any real work project.

This folder was generated by `init-study-project.js`. Everything below assumes
you are standing in this project's root.

---

## What's in here

| Path | What it is |
| --- | --- |
| `.mcp.json` | Two MCP servers: the **knowledge-graph engine** (referenced by ABSOLUTE path) and **gemini-video**. |
| `.claude/settings.json` | KG hooks (auto-recall on each message, search-enforcer, correction detection). |
| `CLAUDE.md` | The study-coach persona + the study loop. |
| `system-design.db` | The knowledge graph. **Created on first run** — not present until then. |
| `mcp-gemini-video/` | The Gemini video MCP server. Has its **own** `package.json` — needs its own `npm install`. |
| `scripts/transcribe.py` | Offline Whisper fallback for transcripts (optional). |
| `lessons/<NN-slug>/` | One folder per lesson, e.g. `lessons/03-cap-theorem/` holding `slides.pdf` + `lesson.mp4`. |

---

## Prerequisites

- **Node.js** (LTS). Powers both MCP servers.
- **The Multi-knowledgeGraph engine present on disk.** `.mcp.json` references the
  engine by its **absolute path** — if you move or rename the engine folder, the
  knowledge-graph server will fail to start until you fix that path.
- **A Google `GEMINI_API_KEY`** — the gemini-video server uses it to watch the
  lecture videos.
- *(Optional)* **Python + `pip install faster-whisper`** — only needed if you
  want the offline `scripts/transcribe.py` transcript fallback. The study loop
  works without it (Gemini reads the video directly).

## Setup (Windows / PowerShell)

### a. Install the Gemini server's own dependencies

The `mcp-gemini-video/` server ships its own `package.json`, separate from the
engine. Install its deps, then return to the project root:

```powershell
cd mcp-gemini-video
npm install
cd ..
```

### b. Set your Gemini API key

```powershell
setx GEMINI_API_KEY "your-key-here"
```

`setx` writes the variable persistently — but it only takes effect in **new**
shells. **Open a new terminal** before continuing, or the gemini-video server
won't see the key.

### c. Drop in a lesson's files

Create a lesson folder named `<NN-slug>` (2-digit number + kebab slug) and place
the slides and the video inside:

```powershell
lessons\03-cap-theorem\slides.pdf
lessons\03-cap-theorem\lesson.mp4
```

### d. *(Optional)* Pre-generate an offline transcript

If you installed `faster-whisper` and want a local transcript instead of relying
on Gemini:

```powershell
python scripts\transcribe.py lessons\03-cap-theorem\lesson.mp4
```

### e. Start the study session

From the project root:

```powershell
claude
```

Claude loads `CLAUDE.md` (the coach persona) and the two MCP servers, and you're
ready to study.

## First run note

The first time the **knowledge-graph** server starts, it downloads the
**~560 MB Qwen3 embedding model**. This happens **once** and is cached
machine-wide — every later run (and every other project using the engine) reuses
the cached model. Expect the first `claude` launch to take a few minutes while
that download completes.

## The study loop

Each lesson follows the same rhythm (the authoritative version lives in
`CLAUDE.md`):

1. **Read the PDF** — Claude reads `slides.pdf` for the lesson's structure.
2. **Watch the video** — the gemini-video server has Gemini watch `lesson.mp4`.
3. **Discuss** — you and the coach talk through the material, surface questions,
   and pin down what actually matters.
4. **Store** — Claude calls `store_knowledge` to record what you learned as
   `principle` (verbatim, with a `quote`), `pattern`, or `inference` nodes.
5. **Connect** — Claude calls `connect_knowledge` to wire causal edges between
   nodes (e.g. `must_precede`, `causes`, `requires_reading`).
6. **Spaced review** — across sessions the coach runs spaced-review quizzes,
   pulling forward nodes that are due so the knowledge sticks.

## This DB is a talent-pool master

`system-design.db` is not throwaway. Once you've studied a topic, it's a
domain **master** you can "hire" into any real work project that uses the same
engine. From the engine's directory, merge it into a project's working brain:

```powershell
node scripts/merge-db.js --into team.db --from system-design.db --tag-domain
```

`--tag-domain` stamps each copied node's `metadata.domain` with the source
filename (`system-design.db`), so you can later filter or prune by domain inside
`team.db`. The merge is idempotent (UUID-level dedup), so re-running is safe.

> **Before merging:** stop this project's knowledge-graph MCP server first.
> `merge-db.js` refuses any `--from` whose `-wal`/`-shm` sidecar is non-empty
> (the sign that its server is still running). Quit `claude` here, then merge.
